# RetainAI v0.4.0-alpha.1 — Multi-Cloud Product Foundation

RetainAI v0.4.0-alpha.1 introduces the first deployable multi-cloud product
foundation while preserving the analytical and explainability capabilities
developed through v0.3.1.

## Highlights

- Public Streamlit dashboard deployed on Google Cloud Run.
- Branded dashboard endpoint at `retainai.hubertronald.dev`.
- FastAPI backend packaged as an AWS Lambda container image.
- Regional API Gateway custom domain at `api.retainai.hubertronald.dev`.
- Dashboard container images stored in Google Artifact Registry.
- Backend container images stored in Amazon ECR.
- Server-side backend token stored through Google Secret Manager.
- Constant-time backend authentication and DynamoDB-backed quota controls.
- Terraform as the active infrastructure-as-code standard.
- Manual application releases from protected `main`.
- AWS OIDC and Google Cloud Workload Identity Federation.
- Independent dashboard and backend image promotion.
- Documentation-only changes excluded from application releases.
- Immutable image tags and successful deployment markers.
- Application delivery separated from Terraform and S3 data operations.

## Architecture boundaries

The browser does not receive backend or AI-provider credentials. Cloud Run calls
the AWS backend from server-side application code. Authentication and quota
validation occur before protected work.

Application releases update only container image references. They do not apply
Terraform or recreate domains, certificates, IAM, secrets, API Gateway, Cloud
Run domain mappings, DynamoDB tables, or data-lake resources.

## Deferred from this prerelease

- Protected S3 data-lake foundation.
- Non-destructive source seed and processing contracts.
- Monitoring and drift reports.
- RAG evidence retrieval.
- Amazon Bedrock and Google Gemini provider enablement.
- Automated infrastructure delivery.

These capabilities remain planned or disabled and are represented separately in
the architecture documentation.

## Upgrade and deployment

This is a prerelease. Application promotion is manual and must be initiated
from the protected `main` branch through the RetainAI multi-cloud application
release workflow.
