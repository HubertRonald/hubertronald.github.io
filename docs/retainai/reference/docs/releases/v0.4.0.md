# RetainAI v0.4.0 — Multi-Cloud Product Foundation

RetainAI v0.4.0 establishes the production-shaped multi-cloud foundation for
the employee-retention Decision Intelligence platform.

## Product foundation

- Reproducible HR data preparation and validation.
- Attrition classification and survival analysis.
- SHAP-based global and local explainability.
- Experiment tracking and model evidence artifacts.
- Controlled prediction and explanation APIs.
- Dashboard-driven decision support.

## Multi-cloud runtime

- Google Cloud Run public dashboard.
- Google Artifact Registry dashboard images.
- Google Secret Manager server-side backend credentials.
- AWS API Gateway branded HTTPS endpoint.
- AWS Lambda container backend.
- Amazon ECR backend images.
- DynamoDB quota enforcement.
- CloudWatch operational logs.
- Terraform-managed AWS and Google Cloud infrastructure.

## Delivery and security

- Manual application promotion from protected `main`.
- Short-lived AWS OIDC and Google Cloud Workload Identity Federation.
- Immutable image references and deployment markers.
- Independent dashboard and backend promotion.
- Documentation-only changes excluded from application release.
- Application delivery separated from Terraform and data operations.

## Data foundation

- Protected and versioned S3 source, curated, artifact, and exchange boundaries.
- Independent data-lake Terraform state.
- Non-destructive source snapshots.
- Run-specific processing outputs.
- Manifest-based dataset promotion.
- Separate processing and cleanup identities.

## Planned evolution

Monitoring, drift-aware validation, RAG evidence retrieval, and provider-neutral
Bedrock and Gemini integrations remain later product capabilities.
